Resend Rapper · Manual

Rails 2 でも Resend.

OpenSSL が古くて TLS 1.2+ を喋れない環境からでも、ローカル LAN 経由で Resend にメールを投げ込むための小さな自前 Web アプリ。送信ログ、 メトリクス、API キー管理、再送、テスト送信まで Web UI でひとそろい。

📥 受け口は HTTP

クライアント側に MTA も TLS も不要。POST /api/v1/send に JSON を投げるだけ。

🔐 出口はモダン TLS

サーバー側で TLS 1.3 を喋り、Resend に転送。古い OpenSSL の問題と無縁。

📊 ログと再送

すべての送信を SQLite に記録。失敗は Web UI からワンクリック再送。

2. しくみ

Rails 2 アプリから ActionMailer 経由で平文 HTTP を投げ、ラッパーが Resend へ橋渡しします。

4. Rails 2 integration

examples/rails2/resend_wrapper_mailer.rb を lib/ に配置し、initializer で有効化。

# config/initializers/resend_wrapper.rb
require File.join(RAILS_ROOT, 'lib', 'resend_wrapper_mailer')

ActionMailer::Base.delivery_method = :resend_wrapper
ActionMailer::Base.resend_wrapper_settings = {
  :endpoint => 'http://10.0.0.5:3000/api/v1/send',
  :api_key  => ENV['RESEND_WRAPPER_API_KEY']
}

これだけで deliver 時に Resend 経由で送られます。Net::HTTP は平文 HTTP を使うため、古い OpenSSL の影響を受けません。

Note. 平文 HTTP は信頼できる内部ネットワーク内でのみ使用してください。インターネット越しに置く場合は Tailscale / VPN / リバースプロキシでの TLS 終端を併用。

8. Security notes

  • クライアント API は 信頼できる内部 NW でのみ 開く。インターネット直公開は不可。
  • API キーは SHA-256 ハッシュ保存。発行時の 1 度しか平文表示されない。
  • Resend API キーは AES-256-GCM (鍵 = MASTER_KEY) で settings に保存。
  • 管理者パスワードは Argon2id でハッシュ。セッションは httpOnly cookie 12h TTL。
  • 定期的にキーをローテートし、使い終わったキーは Revoke。
絶対やらないこと: MASTER_KEY を本番途中で変更しないでください。既存の Resend API キー暗号文が復号できなくなります。
